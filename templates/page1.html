Git Commit:

These commits are snapshots of your whole repository taken at certain periods. You should create new commits often, depending on logical units of change. Over time, commits should explain the tale of your repository's history and how it got to be the way it is now. Commits include a lot of metadata in addition to the contents and message, such as the author, timestamp, and so on. You can use git add. and git commit to add commit changes you make to git. git add: takes a modified file in your working directory and places the modified version in a staging area. git commit: takes everything from the staging area and makes a permanent snapshot of the current state of your repository that is associated with a unique identifier.

How to use Git commit:

  • Step 1:Sign in to your user account on GitHub.
  • Step 2: In the upper right corner, click the + sign icon, then choose New repository. This will take you to a page where you can enter a repository name, description, and choose to initialize with a README file.
  • Step 3: Create a file
  • Step 4: Make a commit with following code: git commit -m "first commit"
  • Step 5: Connect github repository to computer
